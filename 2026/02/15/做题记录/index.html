P13995 【MX-X19-T4】「FeOI Round 4.5」Supernova

诈骗题。

显然 xxx 是单调递增的。据此可以首先判掉一种比较搞笑的 −1-1−1 情况：执行过一次操作1/2后 x>yx>yx>y。

首先弄清楚操作是在干什么：

  • 操作1：将 xxx 末尾一段 01111…1101111\ldots1101111…11 变成 1000…001000\ldots001000…00，然后把 xxx 中 ppp 为 111 的位置也变成 111。
  • 操作2：将 xxx 中 p+1p+1p+1 为 111 的地方变成 111。

上面首先可以挖掘出一个性质：只要进行过一次操作 111 ，那么一定有 p⊂xp\subset xp⊂x。否则一定是一次操作二就能达成条件的。据此可以先判定出一部分 −1-1−1 的情况。

其次发现只要有过一次操作 111 则一定要 x>px>px>p，所以 x+1x+1x+1 一定比 p+1p+1p+1 更优，所以操作2的唯一用处就是第一步可能用一下，接下来一定都用操作1。

然后分析多次的操作1：容易发现由于一定有 p⊂xp\subset xp⊂x，所以考虑把 xxx 和 yyy 中 ppp 为 111 的位置拿掉，剩下的就是相当于 x+1x+1x+1 多少次能变成 yyy。手玩一下发现确实是这样的道理，因为：

  • 考虑一个 ppp 为 111 的位置 sss。容易发现 sss 有影响时当且仅当 sss 后的位置全部为 111，此时 sss 和后面的位置也能构成一个极长 111 段，进位就进到了 s+1s+1s+1 位上，相当于跳过了 sss 这一位，故拿掉 sss 这一位是一样的效果。

然后就可以枚举每一位挨个算了。单次时间复杂度 O(log⁡n)O(\log n)O(logn)，总时间复杂度 O(Tlog⁡n)O(T\log n)O(Tlogn)。

P9352 [JOI 2023 Final] 训猫 / Cat Exercise

经典好题，非常巧妙的思想。

曾经做很多题目的时候，一眼很容易看出来很蠢的在线主席树做法，但是这种题目通常有各方面都更加优秀的离线BIT做法。

这道题目也有类似的想法。

题目一眼看上去就可以发现一个性质：猫到达的塔高度单调递减。

下文令点 uuu 只 pi=up_i=upi​=u 的点 iii。

先钦定题目中的树根为 nnn，显然可以发现堵上点 nnn 后猫的活动范围被限制在了点 nnn 的一个子树内。这给我们一个启示：把猫从点 nnn 开始的每种选择按照前后顺序连接可以形成一棵树，两点之间边权就是两点在原树上的距离，问题的答案就是根到叶子的最长链长。

继续上述步骤，令第一次堵上点 nnn 后猫走到了点 uuu，那么 uuu 下一步可以走到这些地方：

  • uuu 的子树
  • 链 (u,n)(u,n)(u,n) 及其下面挂的一堆子树

我们要建出一棵上述的树就要知道每种选择对应的下一个转移点是哪里。前者容易处理，后者因为要考虑原本的链，子树和接在上面的链的贡献，故处理起来比较麻烦。

这时候怎么办呢？我们发现刚才的做法被困住的点是难以处理 堵上一个点 给答案的贡献。我们换一个角度：考虑把挨个把点堵上的过程倒过来，也就是最开始所有点都被堵上，接下来按照点权 1∼n1\sim n1∼n 逐个解开，也相当于加进图中。由于之前加进图中的点一定都比当前点小，所以直接考虑怎么从之前的点转移过来就可以了。这里的祖先后代关系其实已经不重要，所以拿一个DSU维护一个连通块内最大的点，然后做一个DP就可以了。DP是平凡的。

总的时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

P15245 [WC2026] 二进制

比较牛的，感觉是很人类智慧，遂记之。

一个简单的事实是 y←y×2y\leftarrow y\times 2y←y×2 这个玩意是没用的，给他扔掉就是了。

然后思考一下发现基本有以下两种策略达成目标：

  • 先找到一个最大的 kkk 使得 2kx≤y2^kx\le y2kx≤y，然后若 2kx=y2^kx=y2kx=y 则直接 x←2kxx\leftarrow 2^kxx←2kx，否则x←2k+1xx\leftarrow 2^{k+1}xx←2k+1x，y←y+(x−y)y\leftarrow y+(x-y)y←y+(x−y)
  • 同上，找到一个 kkk，然后通过给 xxx 一系列 +1+1+1 和 ×2\times 2×2 来逼近 yyy，不改变 yyy。

第一种显然是简单的。困难在于第二种。

需要注意的一点是，记 t=(y−2kx)t=(y-2^kx)t=(y−2kx)，第二种的答案其实并非 k+tk+tk+t，因为考虑可以把一些 +1+1+1 放在一些 ×2\times 2×2 前面。

更加需要观察到的一点是，发现上述情况的最优排布是按照 ttt 的二进制摆放，ttt 最低 kkk 位的 111 放在若干个 ×2\times2×2 之间，而更高位只能放在最里层，所以只能硬加。故总答案为 k+⌊t2k⌋+popcount(tmod  2k)k+\lfloor \frac{t}{2^k} \rfloor+popcount(t\mod 2^k)k+⌊2kt​⌋+popcount(tmod2k)。

  • 找到一个 kkk，最初 y←y+ay\leftarrow y+ay←y+a，然后通过给 xxx 一系列 +1+1+1 和 ×2\times 2×2 来逼近 yyy。
